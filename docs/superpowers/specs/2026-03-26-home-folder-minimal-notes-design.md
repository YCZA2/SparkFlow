# 首页 / 文件夹页极简 Notes 风格优化设计

## 背景

SparkFlow 当前移动端首页与文件夹页已经具备完整的信息架构和主流程能力，但视觉层仍偏向“通用列表 + 浮层导航”的默认状态，界面可用但缺少稳定的产品气质。当前产品的核心目标是让知识内容创作者把注意力放在碎片整理和内容创作上，因此本轮不追求更强的视觉冲击，而是收敛为更接近 iOS 备忘录的极简体验。

本次优化范围只覆盖移动端首页和文件夹页的视觉与交互表现，不改变既有信息架构、路由关系、数据源和 local-first 主链路。

## 目标

- 让首页与文件夹页更接近“原生备忘录骨架”，降低界面干扰。
- 保留 SparkFlow 自己的轻度气质，让界面不完全等同系统备忘录。
- 提升内容扫读效率，让用户更容易快速进入文件夹和碎片正文。
- 在不破坏现有功能的前提下，统一两页的布局节奏、留白和状态反馈。

## 非目标

- 不重做全局导航结构。
- 不修改 fragment / folder / script 的业务语义。
- 不新增复杂动画、插画、大面积品牌视觉或营销式 hero 区。
- 不改造录音、导入、生成等主流程逻辑。

## 设计方向

### 整体气质

整体采用“原生 Notes 骨架 + 更精致一点的创作工具气质”。

- 视觉关键词：安静、克制、轻暖、易扫读、可信赖。
- 保留 iOS 风格中的熟悉留白、轻分隔、弱容器感。
- 通过更暖的背景、更稳的层级和更细的强调色控制，建立 SparkFlow 的内容工作台气质。

### 记忆点

记忆点不是强视觉中心，而是“特别安静、内容特别清楚”。用户进入后应先看到内容结构，而不是设计装饰。

## 页面设计

### 首页

首页保持目录页定位，分成三个视觉层级：

1. 顶部信息区
   显示页面标题与总量摘要，保留当前“全部文件夹”的主标题语义，但降低悬浮头部的存在感。标题区应更像系统应用里的自然页眉，而不是独立浮层。

2. 系统入口区
   “全部”“成稿”继续保留，但视觉改成更接近系统分组入口的轻量单元。它们应在页面中足够明确，但不能像功能宫格或营销卡片。

3. 文件夹列表区
   自定义文件夹以统一列表单元展示，弱化厚重卡片，强化标题、数量、副信息与点击区域的一致性。

首页的整体效果应接近“一个被整理过的资料目录页”，而不是堆叠很多独立卡片。

### 文件夹页

文件夹页以内容扫读优先，强调“快速找碎片、快速点进去”：

- 顶部仍保留返回、标题和选择入口，但减少浮层感。
- 列表中的每条碎片更像原生 Notes 条目，而不是独立信息卡。
- 标题、摘要预览、时间、来源的阅读路径要更清晰，默认视线从标题到正文预览自然下落。
- 选择模式只做必要的状态强化，不切换成强操作感界面。
- 底部 AI 操作条延续系统感，保持稳定但不喧宾夺主。

## 视觉规则

### 色彩

- 主背景从纯系统灰白调整为更暖一点的纸感浅色。
- 内容容器保持接近白色，但与背景保留轻微层差。
- 品牌主色仅用于少量交互焦点，例如可点击入口、选中态、关键操作。
- 分隔与边框使用低对比度中性色，避免列表显脏。

### 层级

- 主标题保留较强可读性，但不追求大号视觉标题。
- 副标题、数量、时间和来源采用更克制的层级递减。
- 首页和文件夹页的字重、字号、行高规则尽量复用，减少视觉语言跳变。

### 容器与分隔

- 弱化大面积圆角卡片与明显阴影。
- 更偏向“薄容器 + 轻分隔 + 柔和圆角”的系统式表达。
- 列表项之间要有清晰边界，但边界优先通过间距和浅分隔建立，而不是依赖厚重外框。

### 动效

- 保留必要的页面与选择态过渡。
- 不引入高存在感动效。
- 动效只服务于状态切换的顺滑感，不制造表演感。

## 组件落地策略

### 首页

- `mobile/app/index.tsx`
  调整页面结构与区块层级，弱化当前悬浮渐隐的表演感，重构首页列表节奏。

### 文件夹页

- `mobile/app/folder/[id].tsx`
  调整页面头部、分组标题、选择模式底栏的版式和轻重关系。

### 共用单元

- `mobile/components/FragmentCard.tsx`
  将碎片条目从“轻卡片”收敛为“更接近原生备忘录列表项”的表现。

- 如有必要，可新增一个首页专用的轻量入口行组件或 section 容器组件，但应保持小而清晰，避免引入新的大型抽象层。

### 主题

- `mobile/theme/tokens.ts`
  仅做必要的 token 微调，补足更暖背景、轻分隔和更柔和层次所需的颜色与半径支持。

## 交互约束

- 首页的“全部”“成稿”和文件夹点击逻辑保持不变。
- 文件夹页的多选、下拉刷新、分组展示、进入详情、触发 AI 编导逻辑保持不变。
- 选择模式下仍隐藏 QuickActionBar，退出选择模式后恢复。
- 空态、加载态、错误态沿用现有能力，只做轻量样式对齐，不重写文案策略。

## 测试与验证

本轮以界面回归验证为主：

- 首页能正常展示“全部”“成稿”和自定义文件夹。
- 文件夹页能正常展示 section list 与碎片条目。
- 选择模式下条目选中态和底部 AI 操作条正常工作。
- 深浅色模式下界面都保持可读，且层级关系清楚。
- 页面不因样式调整破坏安全区、下拉刷新和底部遮罩区域。

## 风险与控制

- 如果过度贴近系统备忘录，可能削弱 SparkFlow 的产品辨识度。
  控制方式：只在背景色温、强调色和标题节奏上做轻度品牌表达。

- 如果保留过多现有卡片感，优化后仍会显得普通。
  控制方式：优先调整列表结构、分隔方式和头部层级，而不是只换颜色。

- 如果首页和文件夹页分别单独优化，可能造成视觉规则不一致。
  控制方式：先统一颜色、留白、分隔和条目结构，再分别做页面层级细化。

## 实施范围结论

本次实现以“小范围、高感知”为原则，优先改动首页、文件夹页和碎片条目三个触点，通过较少文件获得更稳定的整体气质提升；不引入结构性重构，不改变业务流程，不扩散到其它页面。

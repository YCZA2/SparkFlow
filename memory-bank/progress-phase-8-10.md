### 8.3 口播稿生成 API ✅

**文件**: `backend/routers/scripts.py`

**实现的端点**:

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/scripts/generate` | POST | 生成口播稿 |

**请求体**:
```json
{
  "fragment_ids": ["id1", "id2", "id3"],
  "mode": "mode_a" // 或 "mode_b"
}
```

**实现逻辑**:
1. 校验 mode 参数 (必须是 mode_a 或 mode_b)
2. 根据 `fragment_ids` 查询碎片 transcript
3. 校验碎片存在性和用户权限
4. 拼接碎片文本（用 `---` 分隔）
5. 根据 `mode` 读取对应 Prompt 文件
6. 调用 LLM 生成口播稿 (temperature=0.7, max_tokens=1500)
7. 写入 `scripts` 表，返回完整口播稿对象

**额外实现**:
- 错误处理：碎片不存在、无转写内容、LLM 调用失败
- 日志记录：关键步骤均有日志输出

### 8.4 口播稿列表 API ✅

**端点**: `GET /api/scripts`

**查询参数**:
- `limit`: 返回数量限制 (默认 20, 最大 100)
- `offset`: 偏移量 (默认 0)

**返回**: 按创建时间降序排列的口播稿列表，包含 id、title、mode、status、is_daily_push、created_at

### 8.5 口播稿详情 API ✅

**端点**: `GET /api/scripts/{script_id}`

**返回**: 单条口播稿完整信息，包含 content、source_fragment_ids 等所有字段

**额外实现的端点**:
- `PATCH /api/scripts/{script_id}`: 更新口播稿状态或标题
- `DELETE /api/scripts/{script_id}`: 删除口播稿

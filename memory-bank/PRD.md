# 【产品名称暂定：灵感编导 AI】MVP版 PRD大纲

## 一、 产品定位与核心价值

**•** **一句话描述**：一款专为知识口播博主打造的，集“碎片语音搜集、个人方法论Agent合稿、原生提词拍摄”于一体的第二大脑工具。

**•** **核心壁垒**：不只是打字机，而是内置了“高阶博主爆款结构”并能“融合多次跳跃碎片灵感”的专属编导。

## 二、 核心业务流 (User Flow)

语音闪记 (多条散落) ➡ 多笔记选中 / AI每日自动聚合 ➡ 双核AI生成口播稿 ➡ 一键发送至提词器 ➡本产品内相机录制 ➡ 保存至手机相册

## 三、 功能模块拆解（开发详细要求）

### 模块1：灵感捕手（首页核心功能）

**•** **大按钮录音**：首页底部放置一个硕大的录音按钮，支持“按住说话/点击说话”。

**•** **录音转写库**：语音结束立刻调用大模型（如阿里云通义听悟、讯飞或微信同款服务）转成文字。

**•** **AI自动摘要与打标签**：不需要用户手动写标题，AI根据语音内容自动生成一句话摘要（如：“关于如何做定位的一点想法”），并自动打上隐形标签，存入【碎片库】。

### 模块2：AI 灵感加工坊（核心亮点）

**•** **手动聚合生成**：在【碎片库】中，用户可以勾选多条（或跳跃的）灵感笔记，点击“交给AI编导”。

**•** **双核Agent引擎**（在生成页面提供两个选项）：

**•** _模式A【导师爆款模式】_：系统预设（你的高阶方法论），强行把散乱的灵感塞进“黄金三秒+痛点+干货+引发互动”的爆款模板里。

**•** _模式B【我的专属二脑】_：根据用户长期积累的词汇习惯、过往发过的老视频文案进行模仿写作。

**•** **每日灵感推盘（Aha Moment）**：

**•** _逻辑_：如果用户昨天记录了>=3条相关联的碎片。今天早上8点，系统后台自动调用大模型，将其融合写成一篇完整的口播稿。

**•** _展示_：通过Push通知提醒用户；用户打开App，首页最显眼处卡片展示：“昨天的4个灵感，已为您写成今日待拍脚本，去看看？”

### 模块3：提词器与拍摄模块（极简闭环逻辑）

**•** **文案流转**：写好的文案详情页，底部大按钮【一键去拍摄】。

**•** **极简相机**：

**•** 调用系统前置/后置摄像头，纯原生画面，**无美颜、无滤镜**（MVP期严控成本）。

**•** 屏幕上方悬浮半透明提词器，文字根据设定速度自动滚动/支持语音跟随滚动。

**•** **本地存储**：拍摄完成确认后，直接调用系统相册API，视频文件保存至手机本地，服务器不留存视频（省带宽和云存储钱）。

### 模块4：我的方法论（底层知识库 - 为未来做准备）

**•** **喂养入口**：允许用户粘贴文字，或者上传TXT/Word，命名为“我的高赞文案”、“我的语言习惯记录”。

**•** **后台架构要求（重点）**：建立 Creator ID 和 Vector Database（向量数据库） 的强绑定。这就为你日后做“知识付费卖Agent”、“允许粉丝订阅某个Creator的方法论”打下了数据隔离的基础。

## 五、 未来功能与底层架构预留说明（技术需知）

_\*\*⚠️ 致开发团队：\*\* 本期虽为 MVP 版本，但产品远期规划会向“创作者SaaS平台”与“多功能拍摄工作流”演进。请在初期架构和数据库设计时，务必为以下功能预留接口、数据字段及UI扩展性。_

### 1\. 双重身份生态：【创作者大厅】

**•** **未来形态**：支持高阶博主将自己的文案素材和逻辑规则打包发布为【专属Agent】（如“美妆种草高转化模型”）。普通用户可在广场订阅/付费调用。

**•** **架构预留**：

**•** **角色权限(RBAC)**：预留 User（普通用户）和 Creator（创作者/模型分享者）标签。

**•** **数据隔离与状态**：Agent模型表预留 Status 字段（私有、公开审批中、已上架）。

**•** **社交网络**：预留用户间的关注关系（Follow）及点赞收藏关联表。

**•** **风控前置**：预留文本/大模型输出内容的敏感词、违规审核接口规范。

### 2\. 跨应用悬浮提词器（打通全域拍摄）

**•** **未来描述**：不仅在App内拍，生成的文案还能变成一个小悬浮窗，悬浮在抖音、剪映、美颜相机等第三方App上提词。

**•** **开发预留要求**：

**•** 初期在App权限管理模块，预留出对“安卓/iOS 悬浮窗权限（画中画/Overlay）”的请求框架。文案数据流要支持剥离出单一的展示组件。

### 3\. 爆款素材解析与拆解器（竞品分析工具箱）

**•** **未来描述**：支持用户扔一个抖音/快手链接进来，系统自动下载去水印视频，提取视频文案，并用AI分析该竞品的“爆款结构”。

**•** **开发预留要求**：

**•** 后端的API网关需预留由外部URL触发任务的异步处理流（Queueing system），因为这类解析任务通常耗时较长，不能阻塞主线程。

**•** 碎片库（灵感库）预留数据来源标识字段（Data Source：语音录入 / 手动输入 / 外部视频解析）。
